HTML

  • The head element contains information about the webpage
  • The meta element contains metadata to tell the browser how to behave/display the webpage
  • The body element contains website content that visible to the user
  • HTML tags tell the browser how to interpret and display the various types of content on a webpage
  • Sectional HTML helps us structure the content of our webpage in a meaningful way

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS best practice is to use an external stylesheet and link it in the head element
  • Using external stylesheets helps keep each different aspect of our website's code separate - rephrase
  • A period preceding a stylesheet element turns it into a selector, allowing us to style multiple HTML elements of the same class

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: pulls down the most recent version in the main branch to local machine
  • git add/commit: stages/pushes changes from local machine to online repo

JavaScript

  • A variable is a named container that allows us to store (and use) data in our code
  • Control flow is the order in which code executes (from top to bottom in JS)
  • Variables must always be declared before they are referenced in functions!
  • Using loops and conditionals allow us to execute and repeat blocks of code according to numeric (e.g. index position) and logical criteria (true/false etc.)
  • Just because we have defined a function, doesn't mean it will execute - we have to call it by its name
